Spécification LFfile

Version : 1.0 (Français)

1. Objectif

LFfile est un format de fichier de traduction léger et très robuste, conçu pour rendre le travail de traduction plus simple, plus sûr et plus rapide.

Les formats existants tels que JSON, XML et YAML présentent souvent les problèmes suivants :

  • Une seule erreur de caractère peut corrompre tout le fichier
  • Syntaxe lourde et sujette aux erreurs
  • Lisibilité réduite

LFfile résout ces problèmes et vise à être un format facile à utiliser aussi bien pour les traducteurs que pour les développeurs.

2. Caractéristiques

2-1. Léger

  • Syntaxe minimale et grande lisibilité
  • Beaucoup moins de lignes que JSON ou XML
  • Malgré sa simplicité, peut contenir beaucoup d’informations, ce qui réduit la taille totale du fichier

2-2. Sûr

  • Chaque ligne est indépendante
  • Une erreur dans une ligne n’affecte pas les autres

2-3. Flexible

  • Pas besoin de guillemets
  • Les variables {} peuvent être écrites telles quelles
  • Les balises HTML comme <br> peuvent être écrites directement

2-4. Résistant aux erreurs

  • Les erreurs n’affectent que la ligne concernée
  • Pas de « corruption totale du fichier » comme avec JSON

2-5. Facile à écrire

  • Moins de caractères nécessaires
  • Flux de travail plus rapide
  • Moins de risques d’erreurs

3. Structure du fichier

LFfile se compose de deux parties : l’en-tête (Head) et le corps (Body).

3-1. En‑tête (Head)

L’en‑tête indique la langue cible.

(lang=Ja)

Règles :

  • Doit suivre ISO 639‑1
  • Majuscule + minuscule (ex. : Ja, En, Fr, De, Ko, Zh)

3-2. Corps (Body)

Format :

<ID> = <texte traduit>

Exemple minimal :

A1 = Hello.

3-3. Règles de lignes

LFfile est basé sur des lignes. Chaque ligne a une signification.

  • En‑tête : 1 ligne
  • Entrée : 1 ligne
  • Commentaire : 1 ligne

Exemple :

(lang=Ja)
A1 = こんにちは。
A2 = Hello.
/This is comments.

4. Comment écrire un fichier LFfile

Créer un LFfile est extrêmement simple.

4-1. Écrire l’en‑tête

(lang=En)
  • Suivre ISO 639‑1
  • Majuscule → minuscule

4-2. Écrire le corps

(1) Obtenir l’ID

Exemple :

<p id="A1">

→ L’ID est A1

(2) Écrire ou traduire le texte

  • {} et <br> peuvent être écrits directement
  • Pas besoin de guillemets

(3) Écrire l’entrée

<ID> = <texte traduit>

Exemple :

A1 = hello.

4-3. Enregistrer le fichier

(1) Enregistrer en .txt puis renommer en .lf

translation.txt → translation.lf

(2) Enregistrer directement en .lf

mytranslation.lf

4-4. Encodage (important)

LFfile doit être enregistré en UTF‑8. Un autre encodage peut provoquer des caractères illisibles.

5. Gestion des erreurs

Chaque ligne étant indépendante, une erreur dans une ligne (sauf l’en‑tête) n’affecte que cette ligne, et le reste du fichier est chargé normalement.

5-1. Indépendance des lignes

  • Une ligne corrompue n’affecte pas les autres
  • La traduction peut continuer

5-2. Traitement des lignes erronées

  • Seule la ligne problématique est ignorée
  • Les autres lignes sont chargées correctement

5-3. Exemple d’erreur (HTML écrit par erreur)

A1 = Hello.
<p id="A1">Welcome</p>
A2 = Bye.

→ Seule la deuxième ligne est invalide. A1 et A2 sont chargées normalement.

5-4. Erreurs faciles à repérer

  • La ligne problématique est immédiatement identifiable
  • Les parseurs peuvent afficher des avertissements

5-5. Comparaison avec d’autres formats

FormatEffet d’une erreur
JSONFichier entièrement corrompu
XMLFichier entièrement corrompu
YAMLErreur d’indentation → fichier corrompu
POCorruption partielle, impact large
LFfileSeule la ligne est ignorée

5-6. Restauration facile

  • Il suffit de corriger la ligne erronée
  • Pas besoin de vérifier tout le fichier

5-7. Avertissement sur l’encodage

Un fichier non enregistré en UTF‑8 peut contenir des caractères illisibles. Utilisez toujours UTF‑8.
