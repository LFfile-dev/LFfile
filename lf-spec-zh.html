LFfile 规范说明

版本：1.0（简体中文）

1. 目的

LFfile 是一种轻量且高可靠性的翻译文件格式，旨在让翻译工作变得更简单、更安全、更高效。

现有格式（如 JSON、XML、YAML）常常存在以下问题：

  • 单个字符错误即可导致整个文件损坏
  • 语法复杂、书写繁琐
  • 可读性较低

LFfile 解决了这些问题，并致力于成为翻译人员与开发者都易于使用的格式。

2. 特点

2-1. 轻量

  • 语法简洁、可读性高
  • 相比 JSON 或 XML 所需行数更少
  • 在保持简洁的同时可记录大量信息，因此在相同翻译量下文件体积比 JSON 更小

2-2. 安全

  • 每一行都是独立的
  • 某一行出错不会影响其他行

2-3. 灵活

  • 无需双引号
  • {} 变量标签可直接书写
  • HTML 标签（如 <br>）可直接书写

2-4. 抗错误能力强

  • 错误仅影响所在行
  • 不会出现类似 JSON 的“全文件损坏”情况

2-5. 易于书写

  • 字符量少
  • 书写速度快
  • 更不易出错

3. 文件结构

LFfile 由两部分组成：头部（Head） 与 主体（Body）。

3-1. 头部（Head）

头部用于指定目标语言。

(lang=Ja)

规则：

  • 必须遵循 ISO 639-1
  • 格式为：首字母大写 + 第二个字母小写（如 Ja、En、Fr、De、Ko、Zh）

3-2. 主体（Body）

格式如下：

<ID> = <翻译文本>

最小示例：

A1 = Hello.

3-3. 换行规则

LFfile 以行为单位，每行具有独立意义。

  • 头部：1 行
  • 每条翻译：1 行
  • 注释：1 行

示例：

(lang=Ja)
A1 = こんにちは。
A2 = Hello.
/This is comments.

4. 如何编写 LFfile

LFfile 的编写非常简单。

4-1. 编写头部

(lang=En)
  • 遵循 ISO 639-1
  • 格式为：大写 + 小写

4-2. 编写主体

(1) 获取 ID

示例：

<p id="A1">

→ ID 为 A1

(2) 编写或翻译文本

  • {} 与 <br> 可直接书写
  • 无需双引号

(3) 写入条目

<ID> = <翻译文本>

示例：

A1 = hello.

4-3. 保存文件

有两种保存方式：

(1) 先保存为 .txt，再改为 .lf

translation.txt → translation.lf

(2) 直接保存为 .lf

mytranslation.lf

4-4. 字符编码（重要）

LFfile 推荐使用 UTF-8 保存。 若未使用 UTF-8，部分字符可能出现乱码。 为避免此问题，请始终使用 UTF-8。

5. 错误处理

LFfile 中每一行都是独立的。 因此，即使某一行（除头部外）出现语法错误， 也只会忽略该行，不影响其他行。

5-1. 行独立性

  • 某一行损坏不会影响其他行
  • 翻译可继续正常进行

5-2. 错误行的处理方式

  • 仅跳过错误行
  • 其余内容正常加载

5-3. 错误示例（误写 HTML）

A1 = Hello.
<p id="A1">Welcome</p>
A2 = Bye.

第二行无效；A1 与 A2 正常加载。

5-4. 易于定位错误

  • 错误行非常容易发现
  • 解析器可选择显示警告

5-5. 与其他格式的比较

格式单行错误的影响
JSON整个文件损坏
XML整个文件损坏
YAML缩进错误导致文件损坏
PO部分损坏，影响范围较大
LFfile仅忽略该行

5-6. 易于修复

  • 只需修复损坏的行
  • 无需检查整个文件

5-7. 编码警告

若未使用 UTF-8 保存，可能出现乱码。 请始终使用 UTF-8。
