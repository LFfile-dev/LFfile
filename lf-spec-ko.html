LFfile 사양서

버전: 1.0 (한국어판)

1. 목적

LFfile은 번역 작업을 더 간단하고, 더 안전하며, 더 빠르게 만들기 위해 설계된 가볍고 높은 안정성을 가진 번역 파일 형식입니다.

JSON, XML, YAML과 같은 기존 형식은 다음과 같은 문제가 있습니다:

  • 한 글자 오류로 전체 파일이 손상될 수 있음
  • 문법이 복잡하여 실수가 발생하기 쉬움
  • 가독성이 낮음

LFfile은 이러한 문제를 해결하며, 번역가와 개발자 모두에게 사용하기 쉬운 형식을 목표로 합니다.

2. 특징

2-1. 가벼움

  • 문법이 단순하고 가독성이 높음
  • JSON이나 XML보다 필요한 줄 수가 훨씬 적음
  • 간결한 문법에도 불구하고 많은 정보를 담을 수 있어 동일한 번역량 기준으로 JSON보다 파일 크기가 작음

2-2. 안전성

  • 각 줄이 독립적으로 처리됨
  • 한 줄의 오류가 다른 줄에 영향을 주지 않음

2-3. 유연성

  • 따옴표가 필요 없음
  • {} 변수 태그를 그대로 작성 가능
  • <br> 같은 HTML 태그도 그대로 작성 가능

2-4. 오류에 강함

  • 오류는 해당 줄에만 영향을 줌
  • JSON처럼 “전체 파일 손상”이 발생하지 않음

2-5. 작성이 쉬움

  • 필요한 문자 수가 적음
  • 작업 속도가 빠름
  • 실수가 적게 발생함

3. 파일 구조

LFfile은 헤더(Head)와 본문(Body)으로 구성됩니다.

3-1. 헤더 (Head)

번역 대상 언어를 지정합니다.

(lang=Ja)

규칙:

  • ISO 639-1을 따름
  • 대문자 + 소문자 (예: Ja, En, Fr, De, Ko, Zh)

3-2. 본문 (Body)

형식:

<ID> = <번역문>

최소 예시:

A1 = Hello.

3-3. 줄바꿈 규칙

LFfile은 줄 단위로 의미를 가집니다.

  • 헤더: 1줄
  • 번역 항목: 1줄
  • 주석: 1줄

예시:

(lang=Ja)
A1 = こんにちは。
A2 = Hello.
/This is comments.

4. LFfile 작성 방법

LFfile은 매우 간단하게 작성할 수 있습니다.

4-1. 헤더 작성

(lang=En)
  • ISO 639-1을 따름
  • 대문자 → 소문자

4-2. 본문 작성

(1) ID 가져오기

예:

<p id="A1">

→ ID는 A1

(2) 번역문 작성

  • {} 및 <br>을 그대로 작성 가능
  • 따옴표 필요 없음

(3) 항목 작성

<ID> = <번역문>

예:

A1 = hello.

4-3. 저장 방법

(1) .txt로 저장 후 .lf로 이름 변경

translation.txt → translation.lf

(2) 처음부터 .lf로 저장

mytranslation.lf

4-4. 문자 인코딩 (중요)

LFfile은 UTF-8로 저장하는 것을 권장합니다. UTF-8이 아닐 경우 글자가 깨질 수 있습니다.

5. 오류 처리

LFfile은 각 줄이 독립적이므로, 헤더를 제외한 줄에서 오류가 발생해도 해당 줄만 무시되고 다른 줄에는 영향을 주지 않습니다.

5-1. 줄 독립성

  • 한 줄의 손상이 다른 줄에 영향을 주지 않음
  • 번역 작업을 계속 진행할 수 있음

5-2. 오류 줄 처리 방식

  • 문제가 있는 줄만 건너뜀
  • 나머지 줄은 정상적으로 로드됨

5-3. 오류 예시 (HTML을 잘못 작성한 경우)

A1 = Hello.
<p id="A1">Welcome</p>
A2 = Bye.

→ 두 번째 줄만 무효. A1과 A2는 정상.

5-4. 오류 발견이 쉬움

  • 문제가 있는 줄을 쉽게 찾을 수 있음
  • 파서는 경고를 표시할 수 있음

5-5. 다른 형식과 비교

형식한 줄 오류의 영향
JSON전체 파일 손상
XML전체 파일 손상
YAML들여쓰기 오류로 손상
PO부분적으로 손상, 영향 범위 큼
LFfile해당 줄만 무시

5-6. 복구가 쉬움

  • 손상된 줄만 수정하면 됨
  • 전체 파일을 검사할 필요 없음

5-7. 인코딩 주의

UTF-8이 아닐 경우 글자가 깨질 수 있습니다. 반드시 UTF-8을 사용하세요.
