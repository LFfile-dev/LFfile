LFfile Spezifikation

Version: 1.0 (Deutsch)

1. Zweck

LFfile ist ein leichtgewichtiges und äußerst robustes Übersetzungsdateiformat, das entwickelt wurde, um Übersetzungsarbeit einfacher, sicherer und schneller zu machen.

Bestehende Formate wie JSON, XML und YAML haben häufig folgende Probleme:

  • Ein einzelner Zeichenfehler kann die gesamte Datei beschädigen
  • Komplexe Syntax, die leicht zu Fehlern führt
  • Geringe Lesbarkeit

LFfile löst diese Probleme und soll ein Format sein, das sowohl für Übersetzer als auch für Entwickler leicht zu verwenden ist.

2. Merkmale

2-1. Leichtgewichtig

  • Minimale Syntax und hohe Lesbarkeit
  • Benötigt deutlich weniger Zeilen als JSON oder XML
  • Trotz kompakter Syntax kann eine große Menge an Informationen gespeichert werden, wodurch die Dateigröße kleiner ist als bei JSON

2-2. Sicher

  • Jede Zeile ist unabhängig
  • Ein Fehler in einer Zeile beeinflusst die anderen nicht

2-3. Flexibel

  • Keine Anführungszeichen erforderlich
  • {}-Variablen können direkt geschrieben werden
  • HTML-Tags wie <br> können ebenfalls direkt geschrieben werden

2-4. Fehlerresistent

  • Fehler betreffen nur die jeweilige Zeile
  • Keine „Gesamtdatei-Korruption“ wie bei JSON

2-5. Einfach zu schreiben

  • Weniger Zeichen erforderlich
  • Schnellerer Arbeitsablauf
  • Geringere Fehlerwahrscheinlichkeit

3. Dateistruktur

LFfile besteht aus zwei Teilen: Kopfbereich (Head) und Rumpfbereich (Body).

3-1. Kopfbereich (Head)

Der Kopfbereich gibt die Zielsprache an.

(lang=Ja)

Regeln:

  • Muss ISO 639-1 entsprechen
  • Großbuchstabe + Kleinbuchstabe (z. B. Ja, En, Fr, De, Ko, Zh)

3-2. Rumpfbereich (Body)

Format:

<ID> = <Übersetzung>

Minimales Beispiel:

A1 = Hello.

3-3. Zeilenregeln

LFfile ist zeilenbasiert. Jede Zeile hat eine eigene Bedeutung.

  • Kopfbereich: 1 Zeile
  • Eintrag: 1 Zeile
  • Kommentar: 1 Zeile

Beispiel:

(lang=Ja)
A1 = こんにちは。
A2 = Hello.
/This is comments.

4. Erstellung einer LFfile-Datei

Das Erstellen einer LFfile-Datei ist sehr einfach.

4-1. Kopfbereich erstellen

(lang=En)
  • ISO 639-1 verwenden
  • Großbuchstabe → Kleinbuchstabe

4-2. Rumpfbereich erstellen

(1) ID ermitteln

Beispiel:

<p id="A1">

→ ID ist A1

(2) Text schreiben oder übersetzen

  • {} und <br> können direkt geschrieben werden
  • Keine Anführungszeichen erforderlich

(3) Eintrag schreiben

<ID> = <Übersetzung>

Beispiel:

A1 = hello.

4-3. Datei speichern

(1) Als .txt speichern und in .lf umbenennen

translation.txt → translation.lf

(2) Direkt als .lf speichern

mytranslation.lf

4-4. Zeichenkodierung (wichtig)

LFfile sollte in UTF‑8 gespeichert werden. Bei anderen Kodierungen können Zeichenfehler auftreten.

5. Fehlerbehandlung

Da jede Zeile unabhängig ist, wird bei einem Fehler (außer im Kopfbereich) nur die betroffene Zeile ignoriert, während der Rest der Datei normal geladen wird.

5-1. Zeilenunabhängigkeit

  • Eine beschädigte Zeile beeinflusst andere nicht
  • Die Übersetzung kann fortgesetzt werden

5-2. Behandlung fehlerhafter Zeilen

  • Nur die fehlerhafte Zeile wird übersprungen
  • Alle anderen Zeilen werden korrekt geladen

5-3. Fehlerbeispiel (HTML versehentlich geschrieben)

A1 = Hello.
<p id="A1">Welcome</p>
A2 = Bye.

→ Nur die zweite Zeile ist ungültig. A1 und A2 werden normal geladen.

5-4. Fehler leicht zu finden

  • Fehlerhafte Zeilen sind sofort erkennbar
  • Parser können optional Warnungen anzeigen

5-5. Vergleich mit anderen Formaten

FormatAuswirkung eines Fehlers
JSONGesamte Datei beschädigt
XMLGesamte Datei beschädigt
YAMLEinrückungsfehler beschädigt Datei
POTeilweise beschädigt, großer Einflussbereich
LFfileNur die Zeile wird ignoriert

5-6. Einfache Wiederherstellung

  • Nur die fehlerhafte Zeile muss korrigiert werden
  • Keine vollständige Dateiprüfung erforderlich

5-7. Kodierungswarnung

Bei nicht UTF‑8 gespeicherten Dateien können Zeichenfehler auftreten. Verwenden Sie immer UTF‑8.
