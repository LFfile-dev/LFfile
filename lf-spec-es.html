Especificación de LFfile

Versión: 1.0 (Español)

1. Propósito

LFfile es un formato de archivo de traducción ligero y altamente robusto, diseñado para hacer el trabajo de traducción más simple, seguro y rápido.

Los formatos existentes como JSON, XML y YAML suelen presentar problemas como:

  • Un solo error de carácter puede dañar todo el archivo
  • Sintaxis excesivamente compleja
  • Baja legibilidad

LFfile resuelve estos problemas y busca ser un formato fácil de usar tanto para traductores como para desarrolladores.

2. Características

2-1. Ligero

  • Sintaxis mínima y alta legibilidad
  • Requiere muchas menos líneas que JSON o XML
  • Aunque es compacto, puede almacenar mucha información, reduciendo el tamaño total del archivo

2-2. Seguro

  • Cada línea es independiente
  • Un error en una línea no afecta a las demás

2-3. Flexible

  • No requiere comillas
  • Las etiquetas de variables {} pueden escribirse directamente
  • Las etiquetas HTML como <br> también pueden escribirse tal cual

2-4. Resistente a errores

  • Los errores afectan solo a la línea correspondiente
  • No ocurre una “corrupción total del archivo” como en JSON

2-5. Fácil de escribir

  • Menos caracteres necesarios
  • Flujo de trabajo más rápido
  • Menor probabilidad de errores

3. Estructura del archivo

LFfile consta de dos partes: Encabezado (Head) y Cuerpo (Body).

3-1. Encabezado (Head)

El encabezado especifica el idioma objetivo.

(lang=Ja)

Reglas:

  • Debe seguir ISO 639-1
  • Una letra mayúscula + una minúscula (ej.: Ja, En, Fr, De, Ko, Zh)

3-2. Cuerpo (Body)

Formato:

<ID> = <text traducido>

Ejemplo mínimo:

A1 = Hello.

3-3. Reglas de líneas

LFfile se basa en líneas. Cada línea tiene significado propio.

  • Encabezado: 1 línea
  • Entrada: 1 línea
  • Comentario: 1 línea

Ejemplo:

(lang=Ja)
A1 = こんにちは。
A2 = Hello.
/This is comments.

4. Cómo escribir un archivo LFfile

Crear un LFfile es extremadamente sencillo.

4-1. Escribir el encabezado

(lang=En)
  • Seguir ISO 639-1
  • Mayúscula → minúscula

4-2. Escribir el cuerpo

(1) Obtener el ID

Ejemplo:

<p id="A1">

→ El ID es A1

(2) Escribir o traducir el texto

  • {} y <br> pueden escribirse directamente
  • No se necesitan comillas

(3) Escribir la entrada

<ID> = <text traducido>

Ejemplo:

A1 = hello.

4-3. Guardar el archivo

(1) Guardar como .txt y renombrar a .lf

translation.txt → translation.lf

(2) Guardar directamente como .lf

mytranslation.lf

4-4. Codificación (importante)

LFfile debe guardarse en UTF‑8. Usar otra codificación puede causar caracteres ilegibles.

5. Manejo de errores

Como cada línea es independiente, si ocurre un error en una línea (excepto en el encabezado), solo esa línea será ignorada, y el resto del archivo funcionará normalmente.

5-1. Independencia de líneas

  • Una línea dañada no afecta a las demás
  • La traducción puede continuar

5-2. Tratamiento de líneas erróneas

  • Solo se omite la línea problemática
  • Las demás líneas se cargan correctamente

5-3. Ejemplo de error (HTML escrito por accidente)

A1 = Hello.
<p id="A1">Welcome</p>
A2 = Bye.

→ Solo la segunda línea es inválida. A1 y A2 se cargan normalmente.

5-4. Errores fáciles de identificar

  • La línea problemática se detecta rápidamente
  • Los analizadores pueden mostrar advertencias

5-5. Comparación con otros formatos

FormatoEfecto de un error
JSONArchivo completamente dañado
XMLArchivo completamente dañado
YAMLError de indentación → archivo dañado
PODaño parcial, impacto amplio
LFfileSolo se ignora la línea

5-6. Fácil recuperación

  • Solo es necesario corregir la línea dañada
  • No hace falta revisar todo el archivo

5-7. Advertencia de codificación

Si no se guarda en UTF‑8, pueden aparecer caracteres ilegibles. Use siempre UTF‑8.
