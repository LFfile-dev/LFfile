LFfile 仕様書

バージョン：1.0（日本語版）

1. 目的

LFfile は、翻訳作業をより簡単・安全・高速にするために設計された、軽量で高い堅牢性を持つ翻訳ファイル形式です。

JSON、XML、YAML などの既存形式には以下のような問題があります：

  • 1 文字のエラーでファイル全体が壊れる
  • 記述が複雑でミスが起きやすい
  • 可読性が低い

LFfile はこれらの問題を解決し、翻訳者と開発者の双方にとって扱いやすい形式を目指しています。

2. 特徴

2-1. 軽量

  • 記述量が少なく、可読性が高い
  • JSON や XML と比べて必要な行数が圧倒的に少ない
  • 記述量が少ない一方で情報量を多く書けるため、同じ翻訳量でも JSON よりファイルサイズが小さくなる

2-2. 安全

  • 各行が独立している
  • 1 行のエラーが他の行に影響しない

2-3. 柔軟

  • ダブルクォーテーション不要
  • {} の変数タグをそのまま書ける
  • <br> などの HTML タグもそのまま書ける

2-4. エラーに強い

  • エラーはその行だけに限定される
  • JSON のような「ファイル全体が壊れる」現象が起きない

2-5. 書きやすい

  • 必要な文字数が少ない
  • 作業が速い
  • ミスが起きにくい

3. ファイル構造

LFfile は ヘッダー（Head） と 本文（Body） の 2 つで構成されます。

3-1. ヘッダー（Head）

翻訳対象の言語を指定します。

(lang=Ja)

ルール：

  • ISO 639-1 に従う
  • 大文字 + 小文字（例：Ja、En、Fr、De、Ko、Zh）

3-2. 本文（Body）

形式：

<ID> = <翻訳文>

最小例：

A1 = Hello.

3-3. 改行ルール

LFfile は行単位で意味を持ちます。

  • ヘッダー：1 行
  • 翻訳エントリ：1 行
  • コメント：1 行

例：

(lang=Ja)
A1 = こんにちは。
A2 = Hello.
/This is comments.

4. LFfile の書き方

LFfile の作成は非常に簡単です。

4-1. ヘッダーを書く

(lang=En)
  • ISO 639-1 に従う
  • 大文字 → 小文字

4-2. 本文を書く

(1) ID を取得する

例：

<p id="A1">

→ ID は A1

(2) 翻訳文を書く

  • {} や <br> をそのまま書ける
  • ダブルクォーテーション不要

(3) エントリを書く

<ID> = <翻訳文>

例：

A1 = hello.

4-3. 保存方法

(1) .txt で保存してから .lf にリネーム

translation.txt → translation.lf

(2) 最初から .lf で保存

mytranslation.lf

4-4. 文字コード（重要）

LFfile は UTF-8 で保存することを推奨します。 UTF-8 以外で保存すると文字化けが発生する可能性があります。

5. エラー処理

LFfile は各行が独立しているため、ヘッダー以外の行でエラーが発生しても、 その行だけが無視され、他の行には影響しません。

5-1. 行の独立性

  • 1 行の破損が他の行に影響しない
  • 翻訳作業を継続できる

5-2. エラー行の扱い

  • 問題のある行だけスキップされる
  • その他の行は正常に読み込まれる

5-3. エラー例（誤って HTML を書いた場合）

A1 = Hello.
<p id="A1">Welcome</p>
A2 = Bye.

→ 2 行目だけ無効。A1 と A2 は正常。

5-4. エラーの発見が容易

  • 問題のある行がすぐに分かる
  • パーサーは警告を出すことも可能

5-5. 他形式との比較

形式1 行のエラーの影響
JSONファイル全体が壊れる
XMLファイル全体が壊れる
YAMLインデントエラーで壊れる
PO部分的に壊れ、影響範囲が広い
LFfileその行だけ無視

5-6. 修復が簡単

  • 壊れた行だけ直せばよい
  • ファイル全体を調べる必要がない

5-7. 文字コードの注意

UTF-8 以外で保存すると文字化けの可能性があります。 必ず UTF-8 を使用してください。
