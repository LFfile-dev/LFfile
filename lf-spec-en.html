LFfile Specification

Version: 1.0 (English Edition)

1. Purpose

LFfile is a lightweight and highly robust translation file format designed to make translation work simpler, safer, and faster.

Existing formats such as JSON, XML, and YAML often suffer from issues such as:

  • A single character error breaking the entire file
  • Excessive syntax requirements
  • Low readability

LFfile solves these problems and aims to be a format that is easy to use for both translators and developers.

2. Features

2-1. Lightweight

  • Minimal syntax and high readability
  • Requires far fewer lines than JSON or XML
  • Despite its compact syntax, it can store a large amount of information, making the total data size smaller than JSON for the same amount of translated text.

2-2. Safe

  • Each line is independent
  • An error in one line does not affect the rest of the file

2-3. Flexible

  • No need for double quotation marks
  • {} variable tags can be written as-is
  • HTML tags such as <br> can also be written directly

2-4. Error-resistant

  • Errors affect only the line where they occur
  • No “entire file corruption” like JSON

2-5. Easy to write

  • Fewer characters required
  • Faster workflow
  • Lower chance of mistakes

3. File Structure

LFfile consists of two parts: the Head and the Body.

3-1. Head

The head specifies the target language.

(lang=Ja)

Rules:

  • Must follow ISO 639-1
  • Two letters: Uppercase + lowercase (e.g., Ja, En, Fr, De, Ko, Zh)

3-2. Body

Format:

<ID> = <translated text>

Minimal example:

A1 = Hello.

3-3. Line Break Rules

LFfile is line-based. Each line has meaning.

  • Head: 1 line
  • Each entry: 1 line
  • Comments: 1 line

Example:

(lang=Ja)
A1 = こんにちは。
A2 = Hello.
/This is comments.

4. How to Write an LFfile

LFfile is extremely easy to create.

4-1. Create the Head

(lang=En)
  • Follows ISO 639-1
  • Uppercase → lowercase

4-2. Create the Body

(1) Obtain the ID

Example:

<p id="A1">

→ ID is A1

(2) Write or translate the text

  • {} and <br> can be written directly
  • No quotation marks required

(3) Write the entry

<ID> = <translated text>

Example:

A1 = hello.

4-3. Saving the File

There are two ways to save an LFfile.

(1) Save as .txt and rename to .lf

translation.txt → translation.lf

(2) Save directly as .lf

mytranslation.lf

4-4. Character Encoding (Important)

LFfile should be saved in UTF-8. If the file is not saved in UTF-8, some characters may become corrupted. To prevent this, always use UTF-8 when creating LFfiles.

5. Error Handling

LFfile uses a structure where each line is independent. Therefore, even if a syntax error occurs in any line other than the head, only that line is ignored, and all other lines work normally.

5-1. Line Independence

  • One broken line does not affect others
  • Translation continues normally

5-2. How Error Lines Are Treated

  • Only the problematic line is skipped
  • The rest of the file loads correctly

5-3. Error Example (HTML mistakenly written)

A1 = Hello.
<p id="A1">Welcome</p>
A2 = Bye.

Only the second line is invalid; A1 and A2 load normally.

5-4. Easy Error Detection

  • The problematic line is immediately identifiable
  • Parsers may optionally show warnings

5-5. Comparison with Other Formats

FormatEffect of a single error
JSONEntire file breaks
XMLEntire file breaks
YAMLIndentation error breaks file
POPartially breaks, wide impact
LFfileOnly that line is ignored

5-6. Easy Recovery

  • Fix only the broken line
  • No need to inspect the entire file

5-7. Encoding Warning

If the file is not saved in UTF-8, some characters may become corrupted. To prevent this, always use UTF-8 when creating LFfiles.
